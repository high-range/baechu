Introduction

This page is designed to test various features of the Webserv HTTP server.

Mandatory Part

Configuration Tests

  • Setup multiple servers with different ports: Port 8080, Port 8081
  • Setup default error page (try to access a non-existing page): 404 Error Page
  • Limit the client body (try to upload a large file)
  • Setup routes in a server to different directories
  • Setup a default file to search for if you ask for a directory: Directory
  • Setup a list of methods accepted for a certain route

Basic Checks

  • GET request: About Page
  • POST request:
  • DELETE request (use a tool like Postman or curl to send a DELETE request)
  • UNKNOWN request (use a tool like Postman or curl to send an unsupported request)
  • Upload and retrieve a file (use a file upload form and then download the file)

CGI Tests

  • Execute CGI script: footer.sh
  • Test CGI with GET method: test_get.py
  • Test CGI with POST method:
  • Test CGI error handling with an erroneous script

Browser Tests

  • Test with a browser and check request/response headers
  • Test wrong URL: Wrong URL
  • List a directory: Directory Listing
  • Test URL redirection: Redirect

Port Issues

  • Test multiple ports and websites: Port 8082, Port 8083
  • Test same port multiple times (check configuration file)
  • Launch multiple servers with common ports

Siege & Stress Test

  • Run stress tests using Siege (ensure availability above 99.5%)
  • Check for memory leaks
  • Ensure no hanging connections
  • Continuous operation without server restart

Bonus Part

  • Working session and cookies system
  • More than one CGI system
