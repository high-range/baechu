Webserv uses non-blocking I/O operations and the kqueue() system call to handle multiple connections efficiently. It is written in C++ 98 and ensures compatibility with major web browsers.

Core Features

  • Support for GET, POST, DELETE methods
  • File uploads
  • Multiple port listening
  • Non-blocking I/O operations
  • kqueue() system call for handling connections

Technical Details

Example configuration:


server {
    listen 80;
    server_name example.com;
    root /var/www/html;
    ...
}

Configuration

Extensive configuration options for managing various tasks.

  • Customizable server settings
  • Support for multiple virtual hosts
  • Flexible directory and file management

Connector

Manages incoming connections and ensures reliable communication.

  • Handles multiple connections concurrently
  • Supports SSL/TLS for secure communication
  • Optimized for performance with non-blocking I/O

Parser

Interprets incoming requests for accurate data processing.

  • Efficiently parses HTTP requests
  • Supports various HTTP methods
  • Handles request headers and body seamlessly

Worker

Processes tasks and handles request responses efficiently.

  • Manages worker threads for handling tasks
  • Ensures balanced load distribution
  • Optimized for minimal latency and high throughput
